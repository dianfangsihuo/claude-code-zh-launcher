# Claude Code 中文启动器一键安装包

这是一个给 Windows 用户使用的 Claude Code 中文启动器。下载 Release 压缩包后，解压并双击 `一键启动DeepSeek.cmd`，填入你自己的 API Key，就可以用 DeepSeek、OpenRouter、硅基流动或其他 OpenAI-compatible 服务开始编程。

## 使用方法

1. 解压整个文件夹。
2. 双击 `一键启动DeepSeek.cmd`。
3. 脚本会安装/更新并打开启动器。
4. 在启动器里填写 URL、API Key 和模型后启动。

DeepSeek 推荐这样填：

```text
URL: https://api.deepseek.com
Model: deepseek-v4-flash
```

优先建议先用 `deepseek-v4-flash` 跑通；需要更强模型再切 `deepseek-v4-pro`。也可以简单填 `flash` 或 `pro`，启动脚本会自动映射。

硅基流动推荐这样填：

```text
URL: https://api.siliconflow.cn/v1
Model: deepseek-ai/DeepSeek-V4-Flash
```

硅基流动也可以简单填 `flash` 或 `pro`，启动脚本会自动映射到 `deepseek-ai/DeepSeek-V4-Flash` 或 `deepseek-ai/DeepSeek-V4-Pro`。

本地中转站推荐这样填：

```text
URL: http://127.0.0.1:3001
Model: gpt-5.5
Provider: openai
```

多数中转站、New API、One API、自建 OpenAI-compatible 网关都选 `Provider: openai`。启动脚本会自动启动本地兼容代理，把 Claude Code 的 Anthropic `/v1/messages` 请求转换成 OpenAI-compatible `/v1/chat/completions` 请求。

OpenRouter 优先推荐这样填：

```text
URL: https://openrouter.ai/api
Model: deepseek/deepseek-v4-flash
Provider: anthropic
API Key: OpenRouter 自己生成的 sk-or-... key
```

OpenRouter 预设会走它的 Anthropic-compatible 接口，少一层本地 OpenAI 格式转换，并使用 Bearer token 方式传递 key。只有手动接 OpenRouter 的 OpenAI-compatible `/api/v1` 路径时，才选择 `Provider: openai`。

智谱 / Z.AI 可以这样填：

```text
URL: https://api.z.ai/api/paas/v4
Model: glm-4.5
Provider: openai
```

或使用国内 BigModel 地址：

```text
URL: https://open.bigmodel.cn/api/paas/v4
Model: glm-4.5
Provider: openai
```

启动器内置了常见预设：DeepSeek、SiliconFlow、智谱/Z.AI、BigModel、DashScope、Moonshot/Kimi、OpenRouter、OpenAI、Gemini OpenAI-compatible、Groq、xAI、Anthropic-compatible direct。预设只是帮你快速填 URL 和模型；任何 OpenAI-compatible 中转都可以选 `Custom / Any OpenAI-compatible` 后手填。

## Thinking / Reasoning

- 默认 `Thinking / Reasoning` 是 `auto`，能力优先：不会主动关闭 DeepSeek V4、SiliconFlow 或 OpenRouter 的思考/推理能力。
- 只有显式选择 `off` 时，脚本才会向支持的上游发送关闭 thinking/reasoning 的参数，用于省成本或绕过供应商兼容问题。
- 选择 `low`、`medium`、`high`、`xhigh`、`max` 会尽量映射到供应商支持的推理强度；DeepSeek 官方 V4 会把 `xhigh/max` 映射到 `max`，普通 high 级别映射到 `high`。
- 带工具调用时，本地代理会保存并回放上游返回的 `reasoning_content`/`reasoning`，避免 thinking 模型在多轮 tool call 后因为缺少推理历史而报错。

只有当你的服务本身就是 Anthropic-compatible `/v1/messages` 接口时，才选择：

```text
Provider: anthropic
```

`Provider: anthropic` 会直连上游，不做 OpenAI 格式转换。

## 脚本会做什么

- 复制项目到 `文档\claude code`。
- 如果电脑没有 Node.js/npm，会执行：

```powershell
winget install -e --id OpenJS.NodeJS.LTS --accept-package-agreements --accept-source-agreements
```

- 在安装目录执行：

```powershell
npm install
```

- 生成 `.env.local` 模板。
- 在桌面创建 `Claude Code.lnk` 快捷方式。
- 启动 Claude Code 时会默认打开桌宠 `Codex-chan for Claude`；它会读取 `~\.claude\projects` 里的最新会话，显示 Claude Code 的近期回复、工具结果和任务进程提示。
- 使用 `Provider: openai` 时，启动脚本会自动启动本地代理，把 Claude Code 请求转成 OpenAI-compatible `/chat/completions` 请求，避免交互模式一直 retry。
- 本地代理默认保留 thinking/reasoning，并把启动器里的 `Thinking / Reasoning` 选项写入 `PROVIDER_REASONING`。
- 默认会把旧 `.env.local` 备份为 `.env.local.bak-时间戳`，再生成新的 DeepSeek 配置，避免继续使用旧中转站或旧 API Key。
- 启动器点击启动时会把当前 URL、API Key、模型写回 `.env.local`，并同步 `ANTHROPIC_AUTH_TOKEN`，避免 Claude Code 进入 `/login`。
- DeepSeek/硅基流动配置只写 `ANTHROPIC_AUTH_TOKEN`，不再写 `ANTHROPIC_API_KEY`，避免 Claude Code 提示 auth conflict。
- 启动器提供 `Hook Safe` 选项，默认启用。它不会关闭全部 hooks，而是把明显会拦普通聊天的 Hookify `UserPromptSubmit` 阻断规则从 `block` 自动降级为 `warn`，并备份原文件。
- 启动器也提供 `Clean / No Hooks` 选项，默认不启用。只有遇到无法修复的 hook 错误时，再勾选 clean 模式临时绕过；clean 会跳过全局记忆、hooks、插件和 auto-memory。

## 注意

- 包里不会包含你的 `.env.local`，不会携带你的 API Key。
- 如果朋友电脑没有 `winget`，请先手动安装 Node.js LTS，再重新运行脚本。
- 主 PowerShell 脚本是 `OneClick-DeepSeek.ps1`，文件名和内容保持 ASCII，兼容 Windows PowerShell 5.1。
- 只需要使用 `一键启动DeepSeek.cmd`。
- 如果不想开桌宠，可以在启动器里取消 `桌宠 / Pet`，命令行加 `-NoPet`，或重开 Claude Code 后用 `/pet off`。
- 桌宠默认低动效；启动器勾选 `Motion` 才会持续播放动画。右键桌宠可以暂停/恢复、变大、变小和关闭。
- 桌宠优先复用 `%USERPROFILE%\.codex\pets\codex-chan\spritesheet.webp`，首次运行会用本机 `ffmpeg` 转成 `assets\pets\codex-chan\spritesheet.png` 缓存。
- 默认不会在命令行要求输入 API Key。
- 如果需要一键修复并验证，可以用 PowerShell 运行 `OneClick-DeepSeek.ps1 -RequireKeyAndVerify`。
- 如果失败，窗口会直接显示日志最后 120 行。
- 同时把 `文档\claude code\DeepSeek-OneClick.log` 发回来即可。
- 如果日志出现 `API Error: 402 Insufficient Balance`，说明 DeepSeek API Key 余额不足，需要换 key 或充值。
- 如果日志显示 `scripts\deepseek-claude-proxy.cjs` 缺失，说明安装目录还是旧文件，重新解压新版包后再双击 `一键启动DeepSeek.cmd`。
